# Blue Archive Data Analytics (総力戦・大決戦データ分析ツール)

ブルーアーカイブ（ブルアカ）の総力戦および大決戦のスコア分布データをビジュアライズし、統計分析を行うためのツールです。
StreamlitによるインタラクティブなWeb画面（GUI）と、OCR・データ管理・グラフ生成を統合したCLIハブツールを提供しています。

---

## 🔧 必要環境・セットアップ

本ツールの実行には Python 3.9 以上が必要です。

### 1. 依存ライブラリのインストール
必要なモジュールをインストールしてください。
```bash
pip install -r requirements.txt
```

---

## 🚀 使い方

### 1. データ管理 Hub（統合CLIツール）の起動 ★推奨
OCRの実行・データ補完・GitHubへのアップロード・グラフ生成など、データ運用に必要な操作をまとめて行えるインタラクティブなメニューです。
```bash
python ocr_hub.py
```

起動後、以下のメニューが表示されます：
```text
==================================================
  ブルアカ データ解析・OCR管理 Hub
==================================================
1: OCRの実行とデータ作成/補完
2: 複数OCR結果の統合
3: 難易度境界順位からのデータ生成・補完
4: 既存データの欠損補完
5: GitHubへの数値データの自動アップデート
6: 可視化グラフの生成
7: 終了
==================================================
```

| メニュー | 内容 |
|---|---|
| **1** | 動画を選択してOCRを実行。新シーズンはWikiから自動検索・登録。OCR後の欠損データを対話型で補完し、そのままGitHubへアップロードする流れに遷移可能。2本目の分割録画を処理する際、同名のParquetが既存の場合は「統合 or 上書き」を確認するプロンプトが自動表示される。 |
| **2** | 既存のParquetファイルをベースに、同イベントの関連動画を追加でOCR・統合する。重複区間のスコア競合は単調減少制約で自動解決し、解決不能な場合はユーザーに確認を求める。補完・上書き保存後にGitHubへアップロード可能。 |
| **3** | 各難易度の一位（`boundary_top`）、総参加者数（`boundary_total`）、主要ボーダー（チナトロ2万/ゴルドロ12万/シルトロ24万位のスコア: `boundary_border`）を手動入力し、間の区間を `missing_interval` としたParquetファイルを一括生成する。登録メニュー内では、間違って登録した手動データを元に戻す「登録データ削除機能（元のOCR実測データがある場合は自動復元）」や、総参加者数が設定された際にそれ以降の不要な超過データを自動で削除・警告する機能も備える。 |
| **4** | 既存のParquetファイルを選択し、欠損している順位のスコアを対話型で手動補完。`missing_interval` 状態の行は自動でスキップされる。完了後にGitHubへアップロード可能。 |
| **5** | `rank_data/` 内の変更されたデータをGitへコミットし、GitHubへプッシュ。 |
| **6** | 指定イベントIDのスコア・タイム分布グラフを画像として生成。 |
| **7** | 終了。 |

> **Note**: Hubツール自身はOCR実行中にCPUをほぼ消費しないため、OCRの並列処理パフォーマンスへの影響は一切ありません。

---

### 2. Streamlit アプリケーションの起動 (GUI)
ブラウザ上でグラフのフィルタリングや分析をインタラクティブに行うことができます。
```bash
streamlit run app.py
```
起動後、自動的にブラウザが開きます（デフォルト: `http://localhost:8501`）。

---

### 3. コマンドラインツールによるグラフ生成 (CLI)
特定イベントのグラフを直接画像として出力したい場合に使用します（Hub の「4」からも実行可能です）。
```bash
# 総力戦 (total_assault_89) のグラフを生成して保存
python run_analysis.py --event total_assault_89

# 大決戦 (grand_assault_34) のグラフを中位ブロック(Mid)で生成して保存
python run_analysis.py --event grand_assault_34 --mode Mid
```

## 📊 データソースとローカルデータについて

### キャッシュの活用とバージョン管理 (推奨)
データは `rank_data/` ディレクトリ配下に `rank_data_{event_id}_{suffix}.parquet` の形式でキャッシュされます。
同じシーズン（イベントID）であっても、後ろに異なるサフィックス（日付時間や `last`）を付与することで、複数の時期のデータを保存可能です。

* **例**:
  * `rank_data_total_assault_90_last.parquet` （最終結果データ）
  * `rank_data_total_assault_90_20260707_0120.parquet` （2026/07/07 01:20 時点の途中データ）

GUI（Streamlitアプリ）上では、同じイベントIDのデータは1つの選択肢に統合され、詳細画面のサイドバーにある**「データ取得時期」**ドロップダウンから自由に表示データを切り替えることができます。

---

### 規約に準拠したデータ取得方法 (推奨アプローチ)
利用規約（ https://bluearchive.jp/terms ）の禁止事項（通信情報の傍受、リバースエンジニアリング、BOT/自動化プログラムの利用など）を厳格に遵守するため、以下の方式によるデータ収集を検討・推奨しています。

* **録画動画からのOCR（画像認識）解析方式**:
  1. ユーザー自身がOSの標準機能（Windows Game BarやOBS等）を使用し、ゲーム内のランキング画面を手動でスクロールする動画（MP4等）をキャプチャします。キャプチャした動画は `OCR/video/` ディレクトリに配置してください。

     **動画ファイルの命名規則**:

     | パターン | 例 | 用途 |
     |---|---|---|
     | `T/Gシーズン_年月日8桁_時間4桁.mp4` | `T90_20260707_0120.mp4` | 総力戦途中経過 |
     | `T/Gシーズン_last.mp4` | `T90_last.mp4` | 最終結果 |
     | `T/Gシーズン_年月日_時間_N.mp4` | `T90_20260707_0120_1.mp4` | 分割録画（重複番号付き）|
     | `T/Gシーズン_last_N.mp4` | `T90_last_2.mp4` | 最終結果の分割録画 |

     同じイベントを複数の動画に分けて録画した場合（例: 1〜210位と200〜500位を別撮り）、末尾に `_1`/`_2` などの重複番号を付けることでファイル名の衝突を回避できます。これらの動画はすべて同一の Parquet ファイルに統合されます（`event_id` に重複番号は含まれません）。

  2. **Hub（推奨）** または直接コマンドで OCR を実行します。Hubからは「1: OCRの実行とデータ作成/補完」を選択してください。
  ```bash
  # Hub から実行（推奨）
  python ocr_hub.py

  # または直接コマンドで実行
  python OCR/video_ocr_parser.py --interval 0.1
  ```
  3. OCR完了後、欠損データが存在する場合は対話型の補完画面が自動的に表示されます。
  4. **同名の Parquet が既に存在する場合**（2本目の分割録画 OCR 時など）、「統合 or 上書き」の確認プロンプトが自動表示されます。`y` を選択すると既存データとマージされます。
  5. 補完完了後、「GitHubへアップデートしますか？」のプロンプトに従い、そのままデータをアップロードできます。
  * ※ ゲームの通信を傍受したりプログラムを改造することなく、また自動スクロール（BOT/マクロ）を行わないため、最も利用規約上の安全性が高い方法です。

---

### 新シーズンの登録について
動画ファイルのシーズン番号（例: `T91_...`）が `common/event_metadata.py` の `EVENT_META` に未登録の場合、OCR実行時に自動で[ブルアカ攻略Wiki](https://bluearchive.wikiru.jp/)へアクセスし、ボス名・開催期間を取得して確認画面を表示します。

* 確認で `y` と入力 → `event_metadata.py` に自動追記・登録されます。
* 確認で `n` と入力、またはWikiに情報がない場合 → キーボードからの手動入力に切り替わります。

## 🆕 最新のアップデート内容

* **大決戦への属性・Torment解放状況の可視化**:
  * ポータル（ボス選択）画面のカードに、大決戦対象となる3つの装甲タイプをバッジとして表示。
  * ボス詳細画面では、Torment難易度が解放されている装甲タイプに `(Torment)` を付記してハイライト表示するよう改善。
  * ポータルおよび詳細画面にフィールド（市街地/屋外/屋内）を判別できる専用の地形画像を右端に追加。
* **検索窓での「双方向スコア・タイム計算機」機能**:
  * 既存の「クリア状況サマリー・プレイヤー検索」の検索窓にスコア（数値）を入れると、最寄りプレイヤーの検索結果に加えて、入力したスコア自体の理論上の「難易度」および「クリアタイム」を `入力スコア` 行として同時表示（スコア ➡ タイム計算）。
  * 検索窓にタイム（例: `Torment 2:17.833`）を入れると、入力タイムに対応する理論スコア（`入力タイム` 行）を同時表示（タイム ➡ スコア計算）。
* **UI/レイアウトの調整**:
  * ポータル画面の各ボスカードにおける下部余白を最適化（`min-height` の調整）。
  * 未記載・欠損しているデータ（データ不足、欠損など）の表示を、全角の「ー」にすべて統一。
  * 各種画面での全角波ダッシュ「～」を半角波チルダ「~」に統一。
